# 규칙 충돌 감사 스킬

## 목적
규칙 파일을 빠르게 점검해 중복/충돌을 제거하고 유지보수 가능한 구조로 정리한다.

## 적용 시점
- 규칙이 많아져 관리가 어려울 때
- 비슷한 규칙이 여러 파일에 반복될 때
- 금지/허용 기준이 섞여 판단이 흔들릴 때

## 감사 절차
1. `.cursor/rules/*.mdc` 파일 목록을 수집한다.
2. 규칙을 공통/도메인/운영 정책으로 분류한다.
3. 중복 항목을 식별하고 단일 소스로 통합한다.
4. 충돌 항목을 식별하고 우선순위를 `금지 > 우선 > 허용`으로 정렬한다.
5. `alwaysApply`와 `globs`를 재조정한다.
6. 레거시 파일은 인덱스 파일로 축소하거나 폐기 계획을 제안한다.

## 결과 형식
- 중복 규칙 목록
- 충돌/모호성 목록
- 권장 파일 구조
- 즉시 적용 순서(1~5단계)

## 주의
- 기존 팀 규칙의 의도를 임의로 삭제하지 않는다.
- 실무 영향이 큰 변경은 사용자 확인 후 적용한다.
