# Laravel 리팩토링 체크리스트

## 목적
Laravel 코드 변경 시 아키텍처 분리 원칙을 빠르게 점검하고, 컨트롤러 중심 로직을 서비스와 Form Request로 정리한다.

## 적용 시점
- 컨트롤러가 길어졌거나 비즈니스 로직이 섞여 있을 때
- 유효성 검사 위치가 혼재되어 있을 때
- 라우트/모델/뷰 책임이 뒤섞였다는 요청이 있을 때

## 실행 절차
1. 변경 범위를 먼저 확정한다. (명시되지 않은 파일은 수정하지 않음)
2. 컨트롤러에서 비즈니스 로직을 식별한다.
3. 식별된 로직을 서비스 메서드로 이동한다.
4. 유효성 검사를 Form Request로 이동한다.
5. 모델에는 관계/스코프만 남기고 복잡한 도메인 로직을 제거한다.
6. Blade는 표시 로직만 남기고 계산/분기 복잡도를 줄인다.
7. 변경 파일 기준으로 린트/기본 동작을 확인한다.

## 산출물 형식
- 변경 이유 1~2문장
- 파일별 변경점 목록
- 위험도(낮음/중간/높음) 및 확인 포인트

## 금지 사항
- 사용자 확인 없이 DB 위험 명령 실행 금지
- 불필요한 예외 처리, 임시 네이밍, TODO 코드 추가 금지
