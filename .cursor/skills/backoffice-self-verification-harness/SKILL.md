# Backoffice Self Verification Harness

## 목적
작업자가 스스로 결과를 검증할 수 있도록 최소/필수 검증 루틴을 표준화한다.

## 적용 시점
- 백오피스 라우트/컨트롤러/뷰/JS를 수정했을 때
- 목록 필터/페이지네이션/폼 저장/복귀 흐름을 변경했을 때
- 공통 레이아웃 또는 공통 스크립트를 건드렸을 때

## 실행 절차 (기본 하네스)
1. **경로 진입 확인**
   - 변경 대상 화면 URL 진입 가능 여부 확인 (404/500/권한 오류 여부)
2. **핵심 시나리오 왕복**
   - 목록 → 상세/수정 → 저장/취소 → 목록 복귀
   - 성공 플래시 메시지/에러 메시지 노출 확인
3. **필터 상태 유지 확인**
   - 검색필터, 표시개수, 페이지 이동 후 수정 진입
   - 목록 복귀 시 query string 유지 여부 확인
4. **유효성 실패 경로 확인**
   - 필수값 누락/형식 오류 입력 후 검증 메시지 확인
   - 버튼 상태(처리중 고착), 포커스 이동, 중복 제출 방지 확인
5. **자산 분리 규칙 확인**
   - Blade 인라인 금지 패턴 정적 검색: `style=`, `onclick=`, `<style`, `<script`
6. **정적 품질 확인**
   - 수정 파일 기준 린트 확인
   - PHP 파일은 최소 구문 검증(`php -l`) 수행

## 변경 유형별 추가 체크
- **CRUD 변경**
  - 생성/수정/삭제 후 목록 수치 및 정렬 반영 확인
- **필터/목록 변경**
  - start/end date, keyword, per_page, pagination 조합 1회 이상 점검
- **공통 스크립트 변경**
  - 백오피스 내 서로 다른 2개 메뉴에서 회귀 확인
- **인증/권한 변경**
  - 로그인/세션 만료/권한 미보유 경로 확인

## 보고 형식 (필수)
1. 실행한 검증 항목
2. 통과/실패 결과
3. 미실행 항목과 사유
4. 남은 위험도(낮음/중간/높음)

## 금지 사항
- 검증 없이 “완료” 보고 금지
- 실패 항목을 숨기거나 추정으로 통과 처리 금지
